## 2. Аудит безопасности и выполненные изменения

### Критические исправления

| Уязвимость | Решение |
|------------|---------|
| **Header injection в send.php** | Из всех полей формы и из `MAIL_FROM` / `MAIL_FROM_NAME` удаляются символы `\r`, `\n`, `%0d`, `%0a` перед подстановкой в заголовки письма. Это предотвращает подмену заголовков и спам через форму. |
| **Конфиденциальные данные** | `config.php` с паролями/учётными данными добавлен в `.gitignore` — не попадает в репозиторий. |

### Дополнительная защита

| Мера | Описание |
|------|----------|
| **Заголовки ответа send.php** | Добавлены `X-Content-Type-Options: nosniff` и `X-Frame-Options: DENY` для снижения рисков MIME-sniffing и встраивания в iframe. |
| **Валидация формы** | Обязательные поля `name` и `contact` проверяются; длина ограничена (`mb_substr`). |
| **Метод запроса** | Обрабатывается только `POST`; на остальные методы возвращается 405. |

### Рекомендации на будущее

- **HTTPS:** обеспечить работу сайта только по HTTPS (настройки хостинга/CDN).
- **Rate limiting:** при росте спама рассмотреть ограничение частоты отправки формы (по IP или по cookie).
- **CSRF:** для форм с критичными действиями можно добавить CSRF-токен; для контактной формы риск умеренный.
- **Логирование:** при необходимости логировать отправки (без персональных данных) для анализа злоупотреблений.
