# Ultra-targeted audit — 3 tasks

## 1) Кейсы: название кейса/компании не должно фиксироваться

**Причина:** Блок `.m-hero` (содержит .m-hero-tag и .m-hero-title — название кейса/компании) имеет `position: sticky; top: 0` и внутри модалки переопределён на `top: var(--nav-height)`. Поэтому при скролле контента кейса этот блок прилипает и двигается как отдельный слой.

**Что сделать:** В модалке сделать заголовок кейса обычным потоковым контентом: `.modal .m-hero { position: relative; top: auto; }`. Убрать класс `.m-hero.is-scrolled` из использования (в JS не вешать его при скролле модалки). Удалить из openCase/closeCase обработчик scroll и снятие/добавление is-scrolled — так и убирается sticky-поведение, и снимается лишний listener (плюс к производительности).

---

## 2) Кнопки: убрать черные заглушки, премиальный вид

**Причина:** Остаточные тёмные тени (`rgba(0,0,0,0.16)`, `0.08`) и отсутствие лёгкой «живости» дают ощущение плоской заглушки.

**Что сделать:** Ещё немного ослабить чёрные тени (0.12, 0.04); добавить очень мягкую, медленную премиальную пульсацию только по бордовому свечению (ключ `button-breath`, 8s, малая амплитуда), без неона и без изменения формы/палитры. Применить к `.btn-minimal` и `.watch-case-btn`. Кнопки шапки не трогать.

---

## 3) Ускорение без визуальных изменений

**Что уже хорошо:** Курсор обновляется по mousemove + один rAF; nav scroll с rAF-throttle; один DOMContentLoaded; модальный scroll listener passive.

**Что сделать:** Удалить обработчик scroll модалки (onModalScroll) и связанную логику is-scrolled — он нужен был только для sticky-панели; после отключения sticky он не нужен, за счёт чего меньше работы при скролле внутри кейса.

---

# Minimal patch plan

1. **CSS**
   - `.modal .m-hero`: заменить sticky на поток: `position: relative; top: auto;` (убрать `top: var(--nav-height)`).
   - Удалить или оставить неиспользуемым `.m-hero.is-scrolled` (можно оставить правило на случай будущего использования вне модалки).
   - Кнопки: ослабить тени до 0.12/0.04; добавить @keyframes button-breath (очень мягкое изменение бордового glow); для .btn-minimal и .watch-case-btn добавить `animation: button-breath 8s ease-in-out infinite`.

2. **JS**
   - В openCase: не добавлять onModalScroll, не сохранять _onModalScroll.
   - В closeCase: не снимать scroll listener и не удалять is-scrolled с .m-hero (логику удалить).

3. **Ограничения**
   - Не менять дизайн, тексты, структуру, контент, медиа. Не трогать шапку и её кнопки.

# Applied changes

**CSS**
- `.modal .m-hero`: `position: relative; top: auto;` — название кейса в модалке ведёт себя как обычный контент, не фиксируется и не двигается.
- Добавлен `@keyframes button-breath` (8s, лёгкая пульсация только бордового свечения).
- `.btn-minimal`: тени ослаблены до 0.12/0.04, добавлена `animation: button-breath 8s ease-in-out infinite`, на hover — `animation: none`.
- `.watch-case-btn`: те же тени и анимация; на hover — `animation: none`.
- Кнопки шапки не изменялись.

**JS**
- В `openCase`: убраны сброс `is-scrolled` у `.m-hero` и добавление обработчика `onModalScroll`.
- В `closeCase`: убраны снятие scroll listener и удаление класса `is-scrolled` у `.m-hero`.

# Changelog

**Fixed**
- Название кейса/компании в модалке больше не фиксируется и не двигается при скролле (position: relative в .modal .m-hero).
- Ощущение чёрной заглушки у кнопок: ослаблены тени, добавлена мягкая премиальная пульсация (button-breath), на hover анимация отключается.

**Preserved unchanged**
- Главная шапка, её кнопки, тексты, структура, контент, изображения, видео, композиция.

**Performance improvements**
- Удалён обработчик scroll модалки и логика is-scrolled — меньше работы при скролле внутри кейса.

**Intentionally not touched**
- HTML, остальные секции, nav, тяжёлые эффекты.

**Why this was the safest fix**
- Один CSS-оверрайд для .m-hero в модалке убирает sticky без смены вёрстки.
- Удаление scroll listener в модалке снижает нагрузку и упрощает код.
- Кнопки: только тени + одна мягкая keyframe-анимация, форма и палитра не менялись.
