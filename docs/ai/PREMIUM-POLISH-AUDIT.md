# Premium/Interaction Audit

## 1) Кнопки — почему всё ещё не премиум

- **Базовое состояние:** Тени 0.1/0.03 уже мягкие, но чёрный слой можно ещё ослабить (0.06/0.02); градиент фона усилить по «воздуху» (0.08→0.03→transparent).
- **Hover:** Сейчас только glow, без лёгкой заливки — нужна очень мягкая смена фона на hover (лёгкий бордовый оттенок), без глухой плашки.
- **Active:** Нет явного :active — добавить лёгкое нажатие (scale или тень).
- **Единый язык:** .btn-minimal и .watch-case-btn уже близки; вынести общие токены в переменные (--btn-border, --btn-shadow-base) и синхронизировать transition/radius.

## 2) Верхняя строка не активируется из кейсов

- **Причина:** При открытой модалке body в position:fixed, скроллится контент модалки — `window.scrollY` не меняется. Класс `scrolled` у nav вешается только при scrollY > 50, поэтому из кейса шапка остаётся в «прозрачном» состоянии и визуально не «включена».
- **Решение:** В openCase принудительно добавлять nav.classList.add('scrolled'), чтобы в кейсе шапка всегда в премиальном стеклянном виде. В closeCase после восстановления scroll вызывать ту же логику, что и у scroll-листера: по текущему window.scrollY добавить или снять 'scrolled'.

## 3) Остаточные лаги / performance

- Дублирующих scroll/mousemove listeners нет; modal scroll listener уже убран.
- Nav при open/close кейса будет запрашиваться через querySelector('nav') дважды за цикл открытия/закрытия — приемлемо, кэшировать не обязательно.
- Курсор и nav уже на rAF + passive. Доп. меры без риска: не добавлять новые лишние слушатели.

## 4) Единый премиальный знаменатель

- Кнопки: один стиль для .btn-minimal и .watch-case-btn (переменные + одинаковый hover/active подход).
- Убедиться, что нигде не осталось тяжёлых чёрных теней (0.4+) у интерактивных элементов вне шапки; при необходимости слегка ослабить только там, где явно «заглушка».

4. **Unification**

# Applied Changes

**CSS**
- :root: добавлены --btn-border, --btn-shadow-subtle, --btn-glow, --btn-inset для единого премиального языка кнопок.

- Кнопки шапки не менялись.

**JS**
- openCase: после открытия модалки вызывается document.querySelector('nav') и nav.classList.add('scrolled') — шапка в кейсе всегда в премиальном стеклянном виде.
- closeCase: после window.scrollTo в requestAnimationFrame пересчитывается состояние nav по window.scrollY (scrolled при > 50) и восстанавливается фокус.

**Performance**
- Новых тяжёлых эффектов и лишних слушателей не добавлено.
