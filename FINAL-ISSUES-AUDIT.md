# Final Issues Audit (точечная доводка)

## A. Audit — причины проблем

### 1. Кнопки: потеря premium feel
- **Причина:** У `.btn-minimal` задано `padding: 14px 0` — нет горизонтального отступа; в модалках (`.m-cta-wrap`) кнопка выглядит «плоской» по бокам. Вертикальное выравнивание текста не задано явно (`inline-flex` + `align-items: center`), из‑за чего надпись может визуально съезжать.
- **Не трогаем:** Цвета, градиенты, border-radius, тени, форму.

### 2. Верхняя строка в кейсах не отображается всегда
- **Причина:** `.m-hero` (заголовок кейса) в потоке модалки; у модалки `overflow-y: auto`. При скролле контента модалки шапка (m-hero) уезжает вверх и пропадает. То есть «верхняя строка» не закреплена.
- **Не проблема:** `.close-case` уже `position: fixed` и всегда виден.

### 3. Верхняя строка в кейсах местами не кликабельна
- **Причина:** Сама по себе кликабельность у `.close-case` есть (z-index 1000003, pointer-events: auto). Если клик «не доходит», возможно, при скролле модалки поверх области кнопки оказывается контент с большим z-index или без `pointer-events: none`. Явно задать `.m-hero` и обёртке верхней строки `pointer-events: auto` и закрепить шапку через `sticky`, чтобы она не уезжала и не перекрывалась контентом.

### 4. Кнопка «Связаться» из кейса ведёт не в тот блок
- **Причина:** Якорь правильный — `#order`, секция одна. После клика: `closeCase()` восстанавливает `scrollY`, затем срабатывает обработчик якоря и делается `scrollIntoView({ block: 'start' })`. Итоговая цель — секция `#order`, но при `block: 'start'` верх секции прижимается к верху viewport, и **фиксированный nav перекрывает заголовок блока**. Воспринимается как «попал не туда» или «не в тот блок».
- **Дубликатов id="order" нет.** Scroll container — окно; sticky header (nav) не учтён при скролле к якорю.

### 5. Overlay / z-index / pointer-events / sticky / overflow / anchor
- **z-index:** modal 1000002, close-case 1000003 — ок.
- **pointer-events:** close-case auto; у .m-hero не задан — задать auto.
- **sticky:** верхняя строка (m-hero) не sticky — при скролле модалки уезжает.
- **overflow:** у .modal overflow-y: auto — из‑за этого шапка скроллится.
- **anchor:** у #order нет `scroll-margin-top` — при скролле к якорю nav перекрывает начало секции.

## B. Minimal Patch Plan

1. **Кнопки (минимально):**
   - `.btn-minimal`: добавить горизонтальный padding `14px 24px` (вместо `14px 0`), явно `display: inline-flex; align-items: center; justify-content: center` для центрирования текста. В контекстах, где нужна полная ширина (например, в карточках цен), оставить `width: 100%` через контекст.
   - Не менять: цвета, тени, border-radius, форму.

2. **Верхняя строка в кейсах:**
   - `.m-hero`: `position: sticky; top: 0; z-index: 2;` и фон как у модалки (например `background: rgba(3,3,3,0.97);`), небольшой `padding-bottom`, `pointer-events: auto`, чтобы шапка всегда была видна при скролле модалки и оставалась кликабельной.

3. **Якорь «Связаться»:**
   - У секции контактов: `#order` или `section#order` задать `scroll-margin-top` (~90–100px), чтобы при скролле к блоку учёт фиксированного nav и заголовок секции был виден.

4. **Ничего лишнего:** не менять тексты, структуру, медиа, глобальный spacing, остальной layout и уже сделанные performance-правки.

## Changelog (минимальные правки)

### Fixed
- **Кнопки:** `.btn-minimal` — padding `14px 24px` (вместо `14px 0`), `display: inline-flex; align-items: center; justify-content: center` для ровного премиального вида текста. Форма, цвета, тени, border-radius не менялись.
- **Верхняя строка в кейсах:** `.m-hero` — `position: sticky; top: 0; z-index: 2; background: rgba(3,3,3,0.97); padding-bottom: var(--space-sm); pointer-events: auto`. Шапка модалки остаётся видимой при скролле и остаётся кликабельной.
- **Якорь «Связаться»:** `section#order { scroll-margin-top: 100px; }` — при переходе по `#order` учёт фиксированного nav, заголовок контактного блока не перекрывается.

### Preserved unchanged
- Тексты, структура, контент, медиа, порядок блоков.
- Стиль кнопок (палитра, градиенты, тени, border-radius 2px).
- Логика модалок, scroll lock, focus, openCase/closeCase.
- Все performance-оптимизации (cursor, scroll throttle, анимации, will-change).
- z-index остальных элементов, общий layout.

### Why this was the minimal safe fix
- Кнопки: только выравнивание и симметричный padding, без смены визуального стиля.
- Верхняя строка: только sticky + фон и pointer-events у существующего блока, без новой разметки.
- Якорь: один свойство scroll-margin-top у целевой секции, без правок JS и без смены id/селекторов.
